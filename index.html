Using a JavaScript package manager (npm)

  • 2013 - Bower was most popular
  • 2015 - Bower was overtaken by npm in popularity
  • 2016 - yarn picked up traction but still uses npm packages under the hood


npm was originally a package manager made specifically for node.js, a JavaScript runtime designed to run on the server, not the frontend.

How to use npm to install moment.js

  • npm init
    This will prompt you several questions in order to make a configuration file called package.json that keeps track of all project info.
  • npm install moment
    This downloads moment's code from the moment.js package into a folder called node_modules/ and updates package.json to keep track of moment.js as a dependency.

Compare js code from momentjs.com to npm downloaded code

  • The Good:can use npm to download and update packages via command line
  • The Bad: still digging through node_modules folder to find location to include it in html.
    But that's because we're on client side. It's much easier on server side... →

You asked...


  • What is npm?
  • What does the command npm init do?
  • What does npm install <package-name> do?

Answers...


  • npm was originally a package manager made specifically for node.js that can install modules published on npmjs.com to extend the functionality of their node apps (but has since expanded its role to be used by front-end devs also)
  • It provides prompts to build a package.json.

    package.json is a file that manages dependencies
  • It downloads the js code/modules in a folder it creates called node_modules.

npm init produces a package.json

This is useful later when sharing a project with others--just need to share package.json (not node_modules/) which others can simply type 'npm install' to install all dependencies.

exports

  • This object is contained in each module and allows you to expose pieces of your code when the module is loaded

  • In Node's CommonJS module implementation, each module is written in a single JavaScript file and has an isolated scope that holds its own variables.

  • The author of the module can expose any functionality through the exports object.
